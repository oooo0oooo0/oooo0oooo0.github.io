## scc

[https://github.com/boyter/scc](https://github.com/boyter/scc)

scc 是一个用于统计代码行数的工具，可以统计代码行数、注释行数、空白行数等。

安装：`brew install scc`

使用：`scc`

``` bash
-h                                    显示帮助
-f, --format string                   输出格式 [tabular, wide, json, json2, csv, csv-stream, cloc-yaml, html, html-table, sql, sql-insert, openmetrics] (default "tabular")
--exclude-dir strings                 排除目录 (default [.git,.hg,.svn])
-x, --exclude-ext strings             排除文件扩展名 (overrides include-ext) [comma separated list: e.g. go,java,js]
-n, --exclude-file strings            排除文件
-M, --not-match stringArray           排除文件和目录匹配正则表达式
-i, --include-ext strings             限制文件扩展名 [comma separated list: e.g. go,java,js]
-o, --output string                   输出文件 (default stdout)
-p, --percent                         输出百分比
```

示例：
``` bash
scc --exclude-dir databases --exclude-file 'test.go' --exclude-file 'rule_infos_level.go' --exclude-file 'pb.go' -p -i go
```

对比`cloc`
``` bash
cloc --exclude-dir=databases --include-lang=go --not-match-f='test\.go$' --not-match-f="rule_infos_level.go" --not-match-f='pb.go$' --fullpath .
```

## micro

[https://github.com/zyedidia/micro](https://github.com/zyedidia/micro)

micro 是一个轻量级的终端编辑器，支持语法高亮、代码补全、主题切换等功能。

安装：`brew install micro`

使用：`micro`

``` bash
micro --version
micro file  打开文件
```

常用快捷键：
``` bash
Ctrl + s 保存
Ctrl + q 退出
Ctrl + e 进入命令模式
Ctrl + w 切换窗口
```

常用命令：
``` bash
keybindings: 给出默认的快捷键列表以及如何重新绑定它们
defaultkeys: 给出更直接的快捷键命令列表和它们的作用
plugins: 解释 micro 的插件系统以及如何创建自己的插件
colors: 解释 micro 的颜色方案和语法高亮引擎以及如何创建自己的颜色方案或添加新的语言到引擎
```
例如，要打开插件的帮助页面，你可以运行 `> help plugins`。

## eza

[https://github.com/eza-community/eza](https://github.com/eza-community/eza)

eza  是一个现代的替代品，用于显示文件列表的命令行程序 ls，它提供了更多的功能和更好的默认设置。

安装：`brew install eza`

使用：`eza`

``` bash
-l, --long                 使用长格式显示文件列表
-G, --grid                 使用网格格式显示文件列表 (默认)
-T, --tree                 树型递归显示目录
-D, --only-dirs            只显示目录
-f, --only-files           只显示文件
-L, --level DEPTH          限制递归深度

-m, --modified             按修改时间排序
-u, --accessed             按访问时间排序
-U, --created              按创建时间排序
--changed                  按修改时间排序
--time-style               时间格式 (default, iso, long-iso, full-iso, relative, or a custom style '+<FORMAT>' like '+%Y-%m-%d %H:%M')
```

## weekly-git-summary

[https://github.com/yinzhenyu-su/weekly-git-summary](https://github.com/yinzhenyu-su/weekly-git-summary)

一个跨平台的 CLI 工具，用于生成 Git 提交记录的周报汇总。支持多种输出格式（文本、JSON、Markdown），自动扫描项目目录并提取指定时间范围内的提交历史。

安装：
``` bash
npm install -g weekly-git-summary
or
npx weekly-git-summary
```

基本用法：
``` bash
# 生成当前目录的本周提交汇总
weekly-git-summary

# 或使用 npx
npx weekly-git-summary
```

常用选项：
``` bash
# 指定目录和时间范围
weekly-git-summary --dir ~/projects --since 2023-01-01 --until 2023-01-31

# 过滤特定作者的提交
weekly-git-summary --author "张三" --since 2023-01-01

# 多作者过滤（OR 关系）
weekly-git-summary --author "张三" --author "李四" --author "王五"

# 使用预设时间范围
weekly-git-summary --time-range this-week
weekly-git-summary --time-range last-month

# 启用传统提交规范分析
weekly-git-summary --conventional --time-range this-week

# 消息模式过滤（支持正则表达式）
weekly-git-summary --message-pattern "feat|fix" --conventional

# 输出不同格式
weekly-git-summary --json
weekly-git-summary --md
weekly-git-summary --html

# 显示帮助信息
weekly-git-summary --help
```
